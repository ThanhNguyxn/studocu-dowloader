### How It Works

The extension handles **3 types of StudoCu documents**:

| Type | Detection | Processing |
|:-----|:----------|:-----------|
| **Image-based** | `<img class="bi">` in page | Canvas conversion to base64 |
| **Text-based** | `<div class="pc">` (paragraph containers) | Direct HTML extraction |
| **Mixed** | Both elements present | Layered rendering (image + text overlay) |

### Image Processing (Cross-Origin Solution)

StudoCu images have `crossorigin` restrictions. We solve this by:

```javascript
// 1. Draw image to canvas
const canvas = document.createElement('canvas');
const ctx = canvas.getContext('2d');
canvas.width = img.naturalWidth;
canvas.height = img.naturalHeight;
ctx.drawImage(img, 0, 0);

// 2. Convert to base64 data URL (bypasses CORS)
const dataURL = canvas.toDataURL('image/jpeg', 0.92);
```

### Text Extraction

Text elements have inline positioning styles that we preserve:

### Print Preview Architecture

The print preview is a new window with:
- **Toolbar** (hidden during print via `@media print`)
- **Page containers** with exact dimensions
- **Layered content**: Image at z-index 1, text at z-index 10

### Cookie Bypass

The blur bypass works by:
1. Getting all cookies for current StudoCu domain
2. Removing cookies that control view limits
3. Reloading page with cleared session

```javascript
chrome.cookies.getAll({ domain: hostname }, (cookies) => {
    cookies.forEach(cookie => {
        chrome.cookies.remove({
            url: `https://${hostname}${cookie.path}`,
            name: cookie.name
        });
    });
});
```
